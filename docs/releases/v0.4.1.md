# ARIL 0.4.1 — Context awareness, Judge readability, and idle health

Point release focused on session context hygiene, clearer Judge comparisons, more
useful cost footers, and a status line that stays accurate while the app is idle.

Builds are signed with Developer ID and **notarized by Apple** — no Gatekeeper
workaround needed.

## Install

1. Download **ARIL-0.4.1.dmg**
2. Open the disk image and drag **ARIL** into **Applications**
3. Launch ARIL (Solo mode starts the bundled gateway)
4. Open **Preferences → General** and paste your OpenRouter API key (if needed)

## What's new in 0.4.1

### Context awareness
- **Per-session context-usage bar** in the sidebar shows how full the conversation
  is relative to the model’s context window.
- ARIL **warns before a session hits the limit** and offers to start a new session.
- Context limits come from the gateway (`/v1/meta/limits`) so the client stays in sync.
- **Fresh session on launch** by default (reduces context exhaustion). Opt into
  reopening your last session in **Preferences → Startup**.
- Empty sessions you never prompt into are discarded on quit; sidebar header is
  **Recent Sessions**.

### Judge mode
- Compare panels **share the full messaging width evenly** and grow taller toward
  the toolbar.
- While Judge results are showing, the **previous transcript is blanked** so text
  no longer bleeds through behind the cards.

### Cost & status
- Reply footers now show OpenRouter’s actual token counts:
  `tokens used [in] / [out]: cost = $x.xxxx`
- Status footer **polls every 20s** (gateway, database, OpenRouter), wakes/restarts
  the Solo gateway on failure, and refreshes when the app becomes active — so idle
  no longer leaves the indicators stuck red.

## Requirements

- **Apple Silicon Mac** (M-series) with **macOS 14** (Sonoma) or newer
- An [OpenRouter](https://openrouter.ai/keys) API key (`sk-or-v1-…`) with a small credit balance
