# ARIL 0.4.3 — Mermaid fix and full-text session search

Point release that makes Mermaid diagrams render reliably in chat, steers models
away from “paste into mermaid.live” apologies, and lets sidebar search find text
inside prompts and replies (not just session titles).

Builds are signed with Developer ID and **notarized by Apple** — no Gatekeeper
workaround needed.

## Install

1. Download **ARIL-0.4.3.dmg**
2. Open the disk image and drag **ARIL** into **Applications**
3. Launch ARIL (Solo mode starts the bundled gateway)
4. Open **Preferences → General** and paste your OpenRouter API key (if needed)

If you already have ARIL 0.4.2+, you can also run **`/update`** in chat.

## What's new in 0.4.3

### Mermaid rendering
- Switched to a classic Mermaid UMD load via a staged `file://` page so diagrams
  actually draw in WKWebView (the previous ES-module CDN approach often failed).
- ARIL always tells the model that diagrams render in-app — no more mermaid.live /
  VS Code detours when you ask for a flowchart.

### Session search
- **Search sessions & content…** matches titles **and** message bodies
  (case-insensitive).
- Matching rows are highlighted; content hits show a short snippet under the title.
- Counter shows `matches/total` while filtering.

## Requirements

- **Apple Silicon Mac** (M-series) with **macOS 14** (Sonoma) or newer
- An [OpenRouter](https://openrouter.ai/keys) API key with a small credit balance
